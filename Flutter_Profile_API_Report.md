# دليل برمجة واجهة الملف الشخصي - فريق الفلاتر (Flutter)

تم إضافة التعديلات اللازمة في الواجهة الخلفية (Backend - Laravel) من أجل عرض وتعديل بيانات الملف الشخصي للمستخدم بكل سهولة. تجدون أدناه تفاصيل مسارات الـ API (Endpoints) المطلوبة لتنفيذ الربط (Integration).

---

## 1. عرض بيانات الملف الشخصي (Get Profile)

يستخدم هذا المسار للحصول على معلومات المستخدم الحالي المسجل للدخول ضمن التطبيق وعرضها في شاشة الملف الشخصي.

- **المسار (URL):** `/api/me`
- **نوع الطلب (Method):** `GET`
- **التوثيق (Auth):** يتطلب إرسال `Bearer Token` في الترويسة (Headers).

### مثال على الرد الناجح (Response - 200 OK):
```json
{
    "data": {
        "id": 1,
        "name": "Ahmed Ali",
        "email": "ahmed@example.com",
        "phone": "+963987654321",
        "role": "user",
        "avatar": "http://localhost:8000/storage/avatars/image.jpg" 
    }
}
```
*(ملاحظة: حقل `avatar` قد يعود بقيمة `null` في حال لم يقم المستخدم برفع صورة شخصية مسبقاً، لذا يرجى أخذ ذلك بعين الاعتبار في الواجهات للـ null safety، ووضع صورة افتراضية (Placeholder)).*

---

## 2. تعديل بيانات الملف الشخصي ورفع الصورة (Update Profile)

يستخدم هذا المسار لتحديث بيانات المستخدم (الاسم، البريد أو رقم الهاتف)، بالإضافة إلى إمكانية تغيير كلمة المرور ورفع وتغيير الصورة الشخصية (Avatar).

- **المسار (URL):** `/api/profile/update`
- **نوع الطلب (Method):** `POST` *(نستخدم POST بدلاً من المعتاد PUT/PATCH لتسهيل إرسال ملفات الصور عبر `multipart/form-data` في بيئة لارافل).*
- **التوثيق (Auth):** يتطلب إرسال `Bearer Token` في الترويسة (Headers).
- **نوع المحتوى (Content-Type):** `multipart/form-data`

### الحقول المُرسلة (Body Parameters):
جميع الحقول أدناه تعتبر **اختيارية (Optional)**. يتم إرسال فقط البيانات التي يتم إجراء التعديل عليها من قبل المستخدم:

| الحقل (Key) | النوع (Type) | الوصف والملاحظات |
|---|---|---|
| `name` | String | الاسم الكامل الجديد للمستخدم. |
| `email` | String | البريد الإلكتروني الجديد. |
| `phone` | String | رقم الهاتف الجديد. |
| `password` | String | كلمة المرور الجديدة (يجب أن لا تقل عن 8 محارف). |
| `password_confirmation` | String | تأكيد كلمة المرور (مُميزة ومطلوبة **فقط** في حال إرسال حقل `password`). |
| `avatar` | File | صورة الملف الشخصي الجديدة (المسموح: jpeg, png, jpg, gif) وبحجم أقصى 2MB. |

### مثال على الرد الناجح (Response - 200 OK):
يعود الـ API بحالة نجاح التعديل بالإضافة لمعلومات المستخدم المُحدثة لتستطيع عكسها مباشرة على الواجهة.
```json
{
    "message": "Profile updated successfully.",
    "data": {
        "id": 1,
        "name": "Ahmed Ali",
        "email": "new.email@example.com",
        "phone": "+963987654321",
        "role": "user",
        "avatar": "http://localhost:8000/storage/avatars/new_image.jpg"
    }
}
```

---

> [!TIP]
> **ملاحظات هامة لمطوري فلاتر (Flutter Guidelines):**
> 1. تأكد من استخدام كلاس `http.MultipartRequest` (أو ما يعادله في `Dio` مثل `FormData`) عند إرسال البيانات إلى `/api/profile/update` لضمان استقبال الـ API لحقل الـ `avatar` كملف (File) بشكل صحيح.
> 2. عند تطبيق تغيير كلمة المرور، يفضل التحقق محلياً في الفلاتر من تطابق حقلي `password` و `password_confirmation` ليتم اختصار إرسال طلب خاطئ إلى الخادم (Server).
> 3. المسار `/api/me` جاهز ويمكن استخدامه كأول طلب ينطلق عند فتح شاشة الملف الشخصي لتعبئة الحقول بالبيانات الأولية للمستخدم.
